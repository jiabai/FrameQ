# FrameQ v0.2.17

FrameQ v0.2.17 是面向个人开发、少量用户和开源工具场景的稳定桌面版本，支持 Windows
x64、macOS Intel 和 macOS Apple Silicon。本版本新增完整桌面多语言与 AI 输出语言控制；
视频、音频和文字稿仍默认在本机处理与保存，SenseVoice Small 模型仍在首次使用时下载。

## macOS 首次打开（下载前请阅读）

macOS DMG 使用 **ad-hoc 签名，未经过 Apple notarization（公证）**。这能验证应用包在
构建后未被修改，但不能建立 Apple Developer ID 身份。因此从 GitHub 下载后，macOS 会在
第一次启动时阻止直接打开。

当前 macOS 的推荐打开方式：

1. 将 FrameQ 拖入“应用程序”并尝试打开一次，然后关闭系统警告。
2. 打开 **系统设置 → 隐私与安全性**。
3. 找到 FrameQ 的拦截提示，点击 **仍要打开（Open Anyway）** 并再次确认。

macOS 14 及更早版本也可在“应用程序”中按住 Control 点击 FrameQ，选择“打开”。如果图形
界面入口不可用，可使用以下终端命令作为后备方式：

```bash
xattr -dr com.apple.quarantine /Applications/FrameQ.app
```

FrameQ 当前不宣称已通过 Apple 公证、App Store 审核或适合静默企业部署。macOS 更新采用
手动下载安装新版 DMG；Windows 继续使用现有签名 updater 元数据。

## 下载选择

- Windows x64：`FrameQ_0.2.17_x64-setup.exe`
- Intel Mac：`FrameQ_0.2.17_x64.dmg`
- Apple Silicon Mac（M 系列）：`FrameQ_0.2.17_aarch64.dmg`

不提供 Linux 安装包，也不提供 Universal macOS 单包。

## 主要改进

- 桌面界面现在内置简体中文、繁体中文和美式英语，可选择跟随系统，并把语言偏好保存在
  本机 app-local 设置中。设置、账户、历史、文字稿、智能提炼、模型下载、进度、错误提示
  和无障碍文案会随语言即时切换。
- 用户确认“要点总结”或“启发灵感”时，FrameQ 会冻结当时实际显示的语言，并要求本次 AI
  输出使用该语言。处理中再次切换界面语言不会改变已经确认的请求。
- AI 输出语言通过严格的桌面-worker 契约传递；缺失、非法或不匹配的字段会安全失败，不会
  猜测旧默认值。桌面端与内置 worker 必须作为同一版本整体升级。
- 处理进度和模型下载状态改为结构化事件，再由桌面端本地化显示；原始 URL、Cookie、凭据、
  文字稿、prompt 和生成内容仍禁止进入进度事件与诊断日志。
- AI 工作区中文名称统一为“智能提炼”，更准确地区分本地文字稿处理与按次确认的云端 AI
  生成。
- GitHub Actions 依赖升级到支持 Node.js 24 的版本，保持 Windows、Intel Mac 和 Apple
  Silicon Mac 三平台发行流程。

## 隐私与安全

- 多语言资源全部随应用离线提供；语言偏好仅保存在设备本地，不进入任务 manifest、历史、
  账户服务或云端 AI 请求。
- AI 仅接收用户确认后的文字稿和固定输出语言语义；界面语言切换不会改变 ASR、字幕选择、
  原始文字稿、缓存或既有历史产物。
- 安装包不包含 ASR 模型权重、LLM key、支付凭据、用户配置、日志、历史任务或用户产物。
- 微信支付仍未启用，`WECHAT_PAY_ENABLED` 继续默认关闭；当前授权方式仍是管理员发放的
  激活码。

## 已知限制

- AI 输出语言是 prompt 约束，供应商仍可能返回其他语言。FrameQ 不会自动检测、翻译或重试，
  也不会因此自动消耗额外 AI Credits。
- macOS 首次启动需要上述 Gatekeeper 手动授权，直到未来采用 Developer ID 签名与 Apple
  notarization。
- macOS 原生系统语言解析与窗口布局需要在本版本 Draft DMG 上记录验收结果；GitHub runner
  的构建、运行时导入和签名完整性检查不能替代真实窗口体验。
- GitHub Releases 在部分网络环境中的下载速度可能较慢；macOS 不提供应用内自动更新。
- 外部视频平台页面或接口可能变化，公开链接下载能力仍受平台可用性影响。
