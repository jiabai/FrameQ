# FrameQ v0.2.16

FrameQ v0.2.16 是面向个人开发、少量用户和开源工具场景的稳定桌面版本，支持 Windows
x64、macOS Intel 和 macOS Apple Silicon。视频、音频和文字稿仍默认在本机处理与保存；
核心 SenseVoice Small 模型在首次使用时下载，不包含在安装包中。

## macOS 首次打开（下载前请阅读）

macOS DMG 使用 **ad-hoc 签名，未经过 Apple notarization（公证）**。这能验证应用包在
构建后未被修改，但不能建立 Apple Developer ID 身份。因此从 GitHub 下载后，macOS 会在
第一次启动时阻止直接打开。

当前 macOS 的推荐打开方式：

1. 将 FrameQ 拖入“应用程序”并尝试打开一次，然后关闭系统警告。
2. 打开 **系统设置 → 隐私与安全性**。
3. 找到 FrameQ 的拦截提示，点击 **仍要打开（Open Anyway）** 并再次确认。

macOS 14 及更早版本也可在“应用程序”中按住 Control 点击 FrameQ，选择“打开”。如果图形
界面入口不可用，可使用以下终端命令作为后备方式：

```bash
xattr -dr com.apple.quarantine /Applications/FrameQ.app
```

FrameQ 当前不宣称已通过 Apple 公证、App Store 审核或适合静默企业部署。macOS 更新采用
手动下载安装新版 DMG；Windows 继续使用现有签名 updater 元数据。

## 下载选择

- Windows x64：`FrameQ_0.2.16_x64-setup.exe`
- Intel Mac：`FrameQ_0.2.16_x64.dmg`
- Apple Silicon Mac（M 系列）：`FrameQ_0.2.16_aarch64.dmg`

不提供 Linux 安装包，也不提供 Universal macOS 单包。

## 主要改进

- 新增严格受控的历史任务永久删除：在明确确认后删除一个受支持的当前任务目录、视频、
  音频、文字稿、AI 产物和对应回听缓存，立即释放磁盘空间；删除不可撤销，文件系统失败
  时会如实报告，不能承诺事务回滚。
- 重组为同一任务下的“本地文字稿”和“AI 整理”两个工作区。本地下载、音频、ASR、校对
  与云端 summary/insights 操作保持独立；只有用户确认后的文字稿片段进入 AI 路径。
- “要点总结”和“启发灵感”现在分别确认、生成和重试；AI 余额统一称为 AI Credits，并
  明确按实际云端 API 调用扣减，不再暗示固定可生成次数。
- 改进历史列表、设置面板、音频播放器、文字稿分段编辑、工具栏和响应式布局的视觉层级、
  键盘交互及可访问性。
- 历史恢复继续由 workflow controller 统一管理；处理、AI retry 或取消期间不能用历史任务
  替换当前任务，晚到的旧回调不能覆盖当前任务。
- 统一 ProcessSupervisor 取消语义：Windows 终止受控进程树，macOS 终止受控进程组；只有
  worker 确认取消后 UI 才进入真实取消终态。
- YouTube 短暂下载失败增加一次受控重试，同时继续保持公开视频、无 Cookie、无登录和
  单视频边界。

## 隐私与安全

- 原始 source URL 只通过受控 stdin 请求进入当前 worker，不进入子进程 argv、环境变量、
  manifest、历史、日志、文字稿、缓存键或 LLM prompt。
- 所有持久化来源身份继续使用 canonical SourceIdentity；仅当前安全 schema v3 历史任务可
  展示、读取、缓存复用、编辑或 retry。
- 不兼容的旧历史目录不会被自动迁移、读取或删除；如需保留或清理，请在产品外自行备份或
  删除。
- 安装包不包含 ASR 模型权重、LLM key、支付凭据、用户配置、日志、历史任务或用户产物。

## 已知限制

- macOS 首次启动需要上述 Gatekeeper 手动授权，直到未来采用 Developer ID 签名与 Apple
  notarization。
- Apple Silicon 安装包具有 GitHub 托管构建、运行时导入和签名完整性验证，但本次发布报告
  仍会区分自动化证据与真实物理设备验收。
- GitHub Releases 在部分网络环境中的下载速度可能较慢；macOS 不提供应用内自动更新。
- 微信支付尚未启用或联调，`WECHAT_PAY_ENABLED` 继续默认关闭；当前可见授权方式仍是管理员
  发放的激活码。
- 外部视频平台页面或接口可能变化，公开链接下载能力仍受平台可用性影响。
