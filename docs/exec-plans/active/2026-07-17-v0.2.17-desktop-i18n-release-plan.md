## Decision Log

- Decision: Use `0.2.17` as a patch release. Rationale: the release packages a completed additive UI
  localization feature and compatible strict worker contract without changing the product's major
  distribution or billing model. Date/Author: 2026-07-17, User + Codex.
- Decision: Define `npm --prefix app run lint` as TypeScript no-emit validation plus the existing
  production i18n-literal boundary check. Rationale: both checks already express current frontend
  quality policy; introducing a new ESLint ruleset immediately before release would create unrelated
  migration risk. Date/Author: 2026-07-17, Codex.
- Decision: Keep tag creation, hosted Draft generation, and public publication outside this execution.
  Rationale: each is an externally visible checkpoint that follows review of the release-prep diff.
  Date/Author: 2026-07-17, User + Codex.
- Decision: New installs and preference recovery use explicit `en-US`; existing saved preferences are
  never migrated, and `system` remains selectable. Rationale: App Settings > Basic must initially
  select `English` without overriding current users. Date/Author: 2026-07-18, User + Codex.
- Decision: Do not tag or broadly publish v0.2.17 until atomic persistence, worker watchdog, and the
  separately tracked server concurrency/operations blocker have acceptance evidence. Rationale:
  clean-machine build success does not prove crash consistency, bounded worker execution, or
  production multi-user correctness. Date/Author: 2026-07-22, User + Codex.
